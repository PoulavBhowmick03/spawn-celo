# Spawn Protocol: Hedge Swarm on Celo

A Darwinian swarm of onchain AI agents that competes to protect stablecoin purchasing power on Celo. Agents run FX and yield strategies across Mento stablecoins (cUSD, cEUR, cREAL) and Aave v3. Every epoch, the fittest agents replicate with mutated parameters and the weakest are culled. Every agent is a registered ERC-8004 identity, every strategy decision is a real mainnet transaction with a published rationale, and every reputation score written onchain is recomputable by anyone from public data.

Built for the Celo Onchain Agents Hackathon (May 22 - June 15, 2026). Spawn Protocol's swarm architecture previously placed 2nd overall at Synthesis and won two tracks at PL Genesis; this is its Celo-native evolution, reframed from yield farming to the problem Celo actually exists for: keeping value stable for people whose local currencies don't.

## Why this fits Celo's mission

Emerging-market users hold Mento stables to escape local currency devaluation, but choosing between cUSD, cEUR, cREAL and idle-vs-lent capital is an active decision most users never make. The Hedge Swarm is that decision-making, automated and evolving: a population of strategies competing in public, with the winning strategies legible onchain. The long-term product is a vault any MiniPay user can deposit into; the hackathon deliverable is the live swarm proving the engine works with real money on mainnet.

## Track mapping (read this, judges)

**Best Agent on Celo.** The swarm is live on mainnet with real positions in Mento and Aave v3. It uses Celo-native features non-trivially: fee abstraction (agents pay gas in the stablecoins they hold and never touch CELO), Mento broker FX, and all three ERC-8004 registries for their intended purposes.

**Most Onchain Transactions.** All transaction volume is organic: swaps, supplies, rebalances, registrations, and reputation writes generated by the epoch loop. There is no transaction emitted for its own sake. The public activity log pairs every tx hash with the strategy rationale that produced it, so activity can be audited, not just counted.

**Highest 8004scan Rank.** Every swarm agent (including retired ones) is a unique ERC-8004 identity on Celo's canonical registries. The orchestrator posts per-epoch reputation feedback derived from a published fitness formula whose inputs are entirely onchain-observable. This is performance attestation, not wash reputation: anyone can recompute every score from Celoscan data using the formula below.

## Fitness formula (public, recomputable)

```
fitness(agent, epoch) = annualize(V_end / V_start) - gas_penalty
V = portfolio value marked in cUSD via Mento broker quotes at epoch boundaries
gas_penalty = total gas paid during epoch, in cUSD, / V_start, annualized
reputation_score = clamp(round(50 + 500 * (fitness - swarm_median)), 0, 100)
```

Epoch boundaries, balances, and gas costs are all readable from Celoscan for each agent address listed on the dashboard.

## Architecture

```
                    ┌──────────────────────────────┐
                    │   Orchestrator (ERC-8004 id) │
                    │  epoch loop · fitness engine │
                    │  reputation writer · logger  │
                    └──────┬───────────────┬───────┘
              evaluate/cull/spawn      feedback per epoch
                           │               │
        ┌──────────────────┼───────┐   ┌───▼─────────────────────┐
        │  Swarm agents (ERC-8004) │   │ ERC-8004 registries     │
        │  MentoFXRotator x N      │   │ Identity · Reputation · │
        │  AaveYielder x N         │   │ Validation (Celo)       │
        │  HedgedCarry x N         │   └─────────────────────────┘
        └────┬──────────────┬──────┘
       Mento Broker     Aave v3 Celo        gas paid in cUSD/USDC
       (cUSD/cEUR/cREAL FX)  (yield)        via CIP-64 fee abstraction
```

Stretch (if present in this deployment): a LedgerForge signal agent selling market signals at sub-cent prices via thirdweb x402; signal-enabled swarm agents pay per call, closing an agent-to-agent economy loop with real x402 payment metrics.

## Quickstart

Prereqs: Node 20+, pnpm, Foundry (for fork tests), a funded Celo wallet (cUSD + USDC, total ≤ $50).

```bash
git checkout celo-hackathon
pnpm install
cp .env.example .env        # fill MNEMONIC, CELO_RPC_URL, budget caps
pnpm test                    # unit: fitness engine, strategy intents
pnpm test:fork               # one full epoch against a Celo mainnet fork
pnpm deploy:celo             # deploy Spawn Protocol contracts to Celo mainnet
pnpm register:agents         # mint ERC-8004 identities, verify on 8004scan
pnpm swarm:start             # goes live; Ctrl-C triggers safe unwind
./run-celo.sh start          # OR: full stack in one command (sleep guard +
                             # x402 oracle + supervised swarm); stop / stop --unwind / status / logs
pnpm swarm:status            # fitness table, balances, links
pnpm report:epoch            # judge-facing epoch report
```

Safety rails baked in: $5 per-agent balance cap, 1% slippage cap, $50 total budget cap, kill switch that unwinds every position to cUSD. All enforced in code, not convention.

### x402 agent-to-agent economy (live)

The signal oracle sells 5-minute-resolution Mento FX momentum + live Aave APYs at **$0.002 USDC per call** over the standard x402 protocol (HTTP 402 → signed `X-PAYMENT` → onchain settlement). Settlement is EIP-3009 `transferWithAuthorization` on Celo-native USDC, submitted by the oracle with cUSD gas — every sale is an auditable USDC transfer (first sale: [`0x7a5135df…50cb`](https://celoscan.io/tx/0x7a5135df3f00a8688e36cbac6cfe5146210ba7420957343c3f19731d8a4050cb)). Buyers are the swarm's own `useSignal` agents, which purchase a signal before each epoch evaluation; the purchased momentum drives their strategy decisions and is credited in their rationale logs.

Retired agents keep their identity and final reputation (e.g. ay-anchor #9244, culled in epoch 1) — honest history is part of the design.

## Honest limitations

This is hackathon-grade software handling real but small funds. Strategies are simple by design; the contribution is the evolutionary loop plus full onchain accountability (ERC-8004 identity, performance-derived reputation, auditable activity), not alpha generation. Do not deposit money you care about.
